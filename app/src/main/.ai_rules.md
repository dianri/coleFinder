# ColeFinder - Protocolo de Arquitectura y Desarrollo (Actualizado 2026)

## 1. Identidad y Estado del Proyecto
- **Propósito:** App nativa para docentes que geolocaliza centros educativos, permitiendo búsquedas por dirección, filtrado y cálculo de distancia en tiempo real.
- **Estado Actual:** MVP Funcional (Carga desde Supabase -> Mapa -> GPS -> Búsqueda -> Filtrado -> BottomSheet).
- **Backend:** Supabase (Alojado en Proxmox: https://api.dianrie.com).

## 2. Stack Tecnológico Principal (Vigente)
- **Lenguaje:** Kotlin 2.0.21 (Target SDK 36) con KSP para procesamiento de símbolos.
- **UI:** Jetpack Compose con Material Design 3 (MD3).
- **Mapas:** Google Maps SDK para Android con la librería `maps-compose`.
- **Inyección de Dependencias:** Hilt (Dagger-Hilt) con `@AndroidEntryPoint` y `@HiltViewModel`.
- **Red:** Supabase SDK (`postgrest-kt` v3.0.3) + Ktor (Android engine).
- **Localización:** Google Play Services Location + Accompanist Permissions.

## 3. Modelo de Datos (Single Source of Truth)
La entidad `Colegio` debe ser `@Serializable` y mapear exactamente:
- `id`: Int (PK)
- `nombre`, `direccion`, `localidad`, `tipo`: String
- `latitud`, `longitud`: Double

## 4. Estructura y Arquitectura Implementada
- **Data Layer:** - `Supabase.client` (Singleton).
    - `NetworkModule` (Hilt): Provee el `SupabaseClient`.
- **UI Layer (MVVM):**
    - `MapState`: Estado único (isLoading, lista de colegios, lista filtrada, userLocation, filtroSeleccionado, cameraPositionState).
    - `MapViewModel`: Gestiona Fetch (Supabase), Geocoder (Búsqueda), Location (GPS) y Distancia (`Location.distanceBetween`).
    - `MapScreen`: Contenedor principal del mapa y componentes UI.

## 5. Componentes de Interfaz y UX
- **Top Bar:** `SearchBar` (M3) para búsquedas gratuitas de dirección mediante `Geocoder`.
- **Mapa:** Marcadores dinámicos con títulos y snippets (Nombre y Tipo).
- **FABs:** Botón de "Mi ubicación" (GPS) y Botón de "Lista" (Panel).
- **ModalBottomSheet:** Panel inferior con `LazyColumn` ordenada por cercanía.
- **Filtros:** Fila de `FilterChips` (Todos, Público, Concertado) con scroll horizontal.
- **Feedback:** Formato de distancia dinámico (metros < 1km > kilómetros) y Badges de colores por tipo.

## 6. Reglas de Desarrollo para la IA
- **Código:** Generar siempre código Kotlin moderno (no usar XML).
- **UI:** Priorizar componentes oficiales de Material 3.
- **Arquitectura:** Toda nueva lógica de datos debe pasar por el ViewModel e integrarse en `MapState`.
- **GPS:** El mapa debe activarse (`isMyLocationEnabled`) solo tras comprobar permisos.
- **Consumo:** Minimizar llamadas a la API de Google Maps (usar Geocoder gratuito y caché de estado).

## 7. Próximos Objetivos (Pendientes)
1. **Persistencia (Room):** Implementar caché local para funcionamiento 100% offline.
2. **Sincronización:** Lógica para actualizar Room desde Supabase solo si hay cambios.
3. **Detalle Avanzado:** Ficha expandida del colegio con contacto y navegación.
4. **Rutas:** Integración de Google Directions para mostrar el camino al centro.

## 7. Instrucciones para la IA
- **Código:** Generar siempre código Kotlin moderno (no usar XML).
- **Errores:** Implementar siempre bloques `try-catch` con estados de error en la UI.
- **Performance:** No realizar cálculos pesados en el hilo principal (Main Thread).
- **API Keys:** No hardcodear claves; usar archivos de configuración o constantes sugeridas.
