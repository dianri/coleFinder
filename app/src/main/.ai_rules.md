# ColeFinder - Protocolo de Arquitectura y Desarrollo (Actualizado Marzo 2026)

## 1. Identidad y Estado del Proyecto
- **Propósito:** App nativa para docentes que geolocaliza centros educativos, permitiendo búsquedas por dirección, filtrado y cálculo de distancia en tiempo real.
- **Estado Actual:** MVP Avanzado. Interfaz limpia con buscador inteligente, mapa numerado por cercanía, gestión de permisos GPS y eliminación de duplicados completada.
- **Backend:** Supabase (Alojado en Proxmox: https://api.dianrie.com).

## 2. Stack Tecnológico Principal (Vigente)
- **Lenguaje:** Kotlin 2.0.21 (Target SDK 36) con KSP.
- **UI:** Jetpack Compose con Material Design 3 (MD3).
- **Mapas:** Google Maps SDK para Android (`maps-compose`).
- **Inyección de Dependencias:** Hilt (Dagger-Hilt).
- **Red:** Supabase SDK (`postgrest-kt` v3.0.3) + Ktor.
- **Localización:** FusedLocationProvider + Geocoder nativo (Gratuito).

## 3. Modelo de Datos (Single Source of Truth)
La entidad `Colegio` debe ser `@Serializable` y mapear:
- `id`: Int (PK)
- `nombre`, `direccion`, `localidad`, `tipo`: String
- `latitud`, `longitud`: Double
- `telefono`: String? (Opcional para Intents de llamada)

## 4. Estructura y Arquitectura Implementada
- **Data Layer:** `SupabaseClient` centralizado y `NetworkModule`.
- **UI Layer (MVVM):**
  - `MapState`: Gestiona `isLoading`, `colegiosConDistancia` (únicos por ID), `userLocation`, `filtroSeleccionado` y `cameraPositionState`.
  - `MapViewModel`: Ejecuta Fetch (Supabase), Geocoder con *debounce* de 300ms para autocompletado, y lógica de ordenación por distancia.
- **Deduplicación:** Obligatorio usar `.distinctBy { it.id }` en el ViewModel antes de actualizar el estado.

## 5. Componentes de Interfaz y UX
- **Buscador (Top):** `SearchBar` (M3) que sustituye a la TopAppBar. Expansible, con sugerencias de direcciones (Geocoder) y colegios en tiempo real.
- **Mapa (Configuración Quirúrgica):**
  - `myLocationButtonEnabled = false` (Botón nativo oculto).
  - `compassEnabled = true` (Brújula visible al rotar).
  - `mapToolbarEnabled = false` (Toolbar de Google oculta).
  - Marcadores numerados del 1 al 50 según proximidad al usuario o punto de interés.
- **FABs:** Botón personalizado de "Mi ubicación" (inferior derecha) y botón de "Lista".
- **Ficha de Detalle:** `ColegioDetailCard` con botones para Intent de Navegación y Llamada. Se oculta al pulsar zona vacía del mapa.

## 6. Reglas de Desarrollo para la IA
- **Código:** Generar siempre Kotlin moderno (no usar XML).
- **UI:** Priorizar componentes oficiales de Material 3 y evitar redundancias visuales.
- **Arquitectura:** Toda lógica de datos debe integrarse en el flujo unidireccional de `MapState`.
- **Rendimiento:** Implementar *try-catch* en lanzamientos de Intents y evitar cálculos pesados en el Main Thread.
- **Consumo:** Priorizar el uso del Geocoder nativo para evitar costes de API externa.
