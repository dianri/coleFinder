# ColeFinder - Protocolo de Arquitectura y Desarrollo (2026)

## 1. Identidad del Proyecto
- **Propósito:** Ayudar a docentes a localizar centros educativos para concursos de traslados basándose en proximidad geográfica.
- **Backend:** Supabase (Alojado en Proxmox: https://api.dianrie.com).
- **Endpoint Principal:** `/rest/v1/colegios`.

## 2. Stack Tecnológico Obligatorio
- **Lenguaje:** Kotlin 2.0+ (Target SDK 36).
- **UI:** Jetpack Compose con Material Design 3 (MD3).
- **Mapa:** Google Maps SDK para Compose.
- **Base de Datos Local (Offline):** Room Database.
- **Red:** Supabase SDK + Ktor (Android engine).
- **Inyección de Dependencias:** Hilt (Dagger-Hilt).

## 3. Modelo de Datos (Entity: Colegio)
La tabla 'colegios' en Supabase y Room debe mapear:
- `id`: Int (PK)
- `nombre`: String
- `direccion`: String
- `latitud`: Double
- `longitud`: Double
- `tipo`: String (Público, Concertado, Privado)
- `localidad`: String

## 4. Reglas de Arquitectura (Clean Architecture)
El código debe estar estrictamente separado en:
1. **Data Layer:** `SupabaseClient`, `ColegioDao` (Room), `ColegioRepositoryImpl`.
2. **Domain Layer:** `Colegio` (Entity), `GetNearbySchoolsUseCase`.
3. **UI Layer:** `NotasViewModel`, `MapScreen`, `SchoolListComponents`.

## 5. Reglas de Interfaz y UX
- **Mapa principal:** Mostrar marcadores numerados del 1 al 50.
- **Side Menu:** Panel lateral derecho ocultable que muestra la lista ordenada por distancia.
- **Buscador:** Barra superior que utiliza `android.location.Geocoder` para búsquedas gratuitas de dirección.
- **Geolocalización:** Botón de "Mi ubicación" para centrar el mapa y buscar centros cercanos.

## 6. Lógica de Sincronización (Sync Strategy)
- La app debe descargar los colegios de Supabase y persistirlos en Room al inicio (si hay red).
- Las búsquedas de proximidad se realizan prioritariamente sobre Room para permitir modo offline.
- Cálculo de distancia: Usar la fórmula de Haversine en el UseCase o ViewModel.

## 7. Instrucciones para la IA
- **Código:** Generar siempre código Kotlin moderno (no usar XML).
- **Errores:** Implementar siempre bloques `try-catch` con estados de error en la UI.
- **Performance:** No realizar cálculos pesados en el hilo principal (Main Thread).
- **API Keys:** No hardcodear claves; usar archivos de configuración o constantes sugeridas.
